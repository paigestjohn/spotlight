02 — Local model

Hosted on your Mac.

The installer uses opencode (opencode.ai, MIT) as the agent harness and Ollama to serve the model locally on 127.0.0.1:11434. opencode reads Spotlight's skills + AGENTS.md natively; native sub-agents handle the investigator/fact-checker pattern. Both install with one command.

The installer uses Pi (pi.dev, MIT) as the agent harness with llama.cpp (via the pi-llama-cpp extension) serving the model locally on 127.0.0.1:8080. Pi lets you browse/load/switch models from inside the harness. Note: Pi has no native sub-agents, so investigator + fact-checker share context.

02 — Runtime

Pick your AI agent.

Spotlight is a skill bundle — it runs inside an AI agent runtime you already have. Subscription runtimes (Claude Code, Gemini, Codex) authenticate via their own login: no API key needed. Only OpenCode requires a provider key (pay-per-token).

~$0.50–2 per investigation, charged per token by the provider's API.

Provider API key is required for OpenCode.

Using Spotlight with Claude Code

  1. First time only: run claude login and sign in with your Claude Max / Pro account — no API key needed
  2. In any terminal, type spotlight — opens Claude Code in your Spotlight repo with all skills loaded
  3. Tell it: "Start a Spotlight investigation on <your lead>"
  4. Claude walks you through Preflight → Brief → Methodology → 5 execution cycles → Gate 1 review → Ingestion into your vault

Also works in the Claude Desktop app: open the Spotlight folder, Claude sees CLAUDE.md automatically. Covered by Claude Max / Claude Pro — no per-investigation cost.

Using Spotlight with OpenCode

  1. Paste your provider key above — OpenCode is pay-per-token and needs an API key for OpenRouter / Fireworks / Together
  2. In any terminal, type spotlight — opens OpenCode in your Spotlight repo with all skills loaded
  3. First time: type /model and pick a strong default for your provider
  4. Tell it: "Start a Spotlight investigation on <your lead>"
  5. OpenCode walks you through Preflight → Brief → Methodology → 5 execution cycles → Gate 1 review → Ingestion into your vault

Recommended models: OpenRouter → z-ai/glm-5.1; Fireworks → qwen3p6-plus; Together → a Qwen3.6 variant. Cost is per-token.

05 — Vault Required

Your investigative memory.

Spotlight archives verified findings into a Markdown vault as structured knowledge — entities, methodology, tools — with wikilinks and YAML frontmatter. Over time, the vault becomes your investigative memory.

The installer brew-installs Obsidian if it isn't already on your Mac, then launches it so you can complete one small step below.

One manual step (Obsidian 1.12+)

Obsidian's CLI toggle can't be automated — you must click it once after the app launches:

  1. The installer opens Obsidian for you
  2. In Obsidian → Settings → General → Advanced → toggle Command Line Interface ON
  3. That's it. Spotlight detects the obsidian command on its first run and prompts you again if it's missing.

If Obsidian is already installed and the CLI is on, the installer skips this step. If you haven't opened Obsidian in a while, verify you're on 1.12 or newer (About menu).

The installer downloads Tolaria on macOS if it isn't already installed, then opens it. Spotlight writes standard Markdown files with YAML frontmatter, so the same vault remains readable from Tolaria, Git, and your agent tools.

Final install path: ~/Documents/Spotlight
